Un site rapide est récompensé. Et le choix du framework décide à 80% si votre site sera rapide ou lent. Pourquoi WordPress souffre sur les CWV

WordPress charge plusieurs centaines de Ko de CSS et JavaScript par défaut — avant même d'activer un thème ou un plugin. Elementor seul ajoute 300 à 500 KB. La plupart des sites WordPress arrivent à 1 à 2 MB de ressources à charger avant d'afficher quoi que ce soit.

Ce n'est pas un problème de WordPress en soi — c'est un problème d'architecture. WordPress est conçu pour la flexibilité, pas pour la performance. Il existe des sites WordPress rapides, mais ils nécessitent une expertise poussée et un investissement continu pour maintenir les performances.

Pourquoi Astro performe naturellement

Astro génère des fichiers HTML statiques purs. Pas de JavaScript côté client par défaut. Le navigateur reçoit du HTML brut, l'affiche immédiatement, et ne charge du JavaScript que pour les composants interactifs spécifiques. C'est ce qu'on appelle l'architecture "Islands".

Résultat : le LCP (chargement du premier élément visible) est naturellement sous 1 seconde. Les Core Web Vitals sont verts sans optimisation particulière.

Ce que ça change concrètement pour votre référencement

  • Un site rapide obtient un meilleur classement sur les mêmes mots-clés qu'un site lent (toutes choses égales par ailleurs)
  • Un LCP < 2,5s réduit le taux de rebond de 32% en moyenne (source : Google)
  • Un score Lighthouse 90+ est un signal de confiance visible dans la Search Console
  • Les crawlers Google indexent les pages statiques plus rapidement et plus souvent
